feat(voice-engine): Sprint 7.16 — Adaptive Voice Engine MVP Phase 1

Full TypeScript port of the 4 core Python modules from
Helm SEO/helm-adaptive-voice-engine/ + integration into the
content generation pipeline. Schema applied to prod via
scripts/hotfix-sprint-7.16-apply.mjs before this commit landed.

Architecture (1:1 with the Python source)
-----------------------------------------
  lib/voice-engine/types.ts             ← client_context.py
    - 7 enums (Platform, ContentType, Dimension, MaturityStage,
      FeedbackTier, Volatility, SignalSource)
    - 8 records (Signal, Override, AuditEntry, WeightedPost,
      PerformanceProxy, BrandBibleVoiceEngine, PlatformSlots,
      ClientContext)
    - MATURITY_STAGE_CONFIG, FEEDBACK_TIER_WEIGHTS,
      DIMENSION_VOLATILITY, MAGNITUDE_CAP_MULTIPLIER tables
    - Free-function ports of Python methods so all records stay
      JSON-round-trippable.

  lib/voice-engine/loader.ts            ← DB persistence layer
    - loadClientContext: get-or-create, seeds from the project
      BrandBible on first touch.
    - saveClientContext: full JSONB rewrite (no diff-patch; the
      blob is bounded and rewrite is cheap vs the Opus call that
      produced the mutation).
    - logAudit / appendAuditEntryByProject: normalized audit log
      writes for operator queries.

  lib/voice-engine/diff-classifier.ts   ← diff_classifier.py
    - 7 heuristics (banned vocab single + phrase, hook length,
      emoji count, hashtag count, CTA shift, paragraph length).
    - Same confidence calibration as the Python source.
    - LLM-batch stub preserved for Phase 1.5.

  lib/voice-engine/feedback-loop.ts     ← feedback_loop_service.py
    - processSignals: gate by stage / cool-down / weighted
      volume; derive new override; apply magnitude cap; emit
      audit entry.
    - rollbackOverride / recordTieredFeedback /
      incrementPostCount.
    - All MVP-deferred slots (circuit breaker, auto-rollback,
      stale decay, shadow mode, performance reweighting)
      preserved as comment markers.

  lib/voice-engine/prompt-builder.ts    ← prompt_builder.py
    - buildAdaptivePrompt: stacks PROMPT_COMPOSITION_RULES (from
      Sprint 7.13 platform-tone, reused not duplicated) +
      dynamic CLIENT CONTEXT block + pain point + content-type
      rules + content-type examples + platform tone.
    - parseOverrideLog: strips + extracts override_log tags from
      model output.

Schema (lib/db/schema.ts + apply script)
----------------------------------------
  client_contexts: per-project (unique on project_id), JSONB-
    heavy for the bounded learning state.
  voice_engine_audit_log: normalized row-per-state-change with
    indexes on (action, dimension, user_id, created_at) so the
    operator grep queries stay cheap.

Endpoints (3 new, none replacing existing surfaces)
---------------------------------------------------
  POST /api/voice-engine/record-edit
    Body: projectId, platform, contentType, postId, original,
          edited, feedbackTier?
    Classifies the diff, applies tier weight when present, runs
    processSignals, persists context + audit entries.

  POST /api/voice-engine/record-publish
    Body: projectId, platform, contentType, postId, text,
          qualityScore?
    Appends to voiceFingerprint (cap 30), increments post_count
    (drives maturity stage), writes a post_published audit row.

  POST /api/voice-engine/rollback
    Body: projectId, platform, dimension, reason?
    Authorization: Bearer HELM_OPERATOR_KEY
    Operator-only manual override removal + audit row.

Integration in /api/ai/generate-structured
------------------------------------------
  1. Load ClientContext once at the top of POST (best-effort —
     a load failure falls through to the Sprint 7.13 prompt path).
  2. Per-template loop: when platform + content_type are in the
     taxonomy AND context loaded, use buildAdaptivePrompt; else
     use buildGenerationPrompt (Sprint 7.13 fallback).
  3. After Opus: parseOverrideLog strips + captures the model
     override_log tag block BEFORE JSON.parse. Each record
     becomes a model_applied_override audit row.

Doesn't change
--------------
  - Sprint 7.14 Voice + Performance Memory blocks still flow
    into the system prompt. The Voice Engine adds a NEW layer;
    it does not replace the old one.
  - Sprint 7.13 platform-tone is the static scaffold —
    architectural invariant from the brief: NEVER mutates.
  - Legacy /api/ai/generate-post pipeline untouched.

Deferred to Phase 1.5+ (stubs in place)
---------------------------------------
  LLM batch diff classifier, cross-platform fingerprint, stale
  decay sweeper, shadow mode, performance reweighting,
  exploration sampling, cohort comparison. All slots in the
  data model so adding them later does not need a migration.
